## 1. 云原生交付模块（Docker-Compose 一键部署）

### 做什么
让评委或任何人用一条命令就能把整个《密盾》系统跑起来。

### 怎么做
写一个 `docker-compose.yml`，把 Go API 服务和 Nginx 反向代理打包在一起。

### 包含什么
- Go API 容器（midun-server）
- Nginx 容器（反向代理 + HTTPS）
- 健康检查：自动检测服务是否正常运行

### 当前进度
- ✅ Dockerfile 已完成
- ⬜ docker-compose.yml 待编写

## 2. 可信签发模块（VC 签发）

### 做什么
模拟权威机构，给用户签发一个"数字凭证"，证明用户的身份属性（比如年龄）。

### 怎么做
- 用户提交身份信息
- 后端用 ECDSA 算法对信息签名
- 返回一个 JWT 格式的凭证

### 接口
- 路径：`POST /v1/vc/issue`
- 请求：`{ "user_id": "xxx", "attributes": {"age": 20} }`
- 响应：`{ "credential": "签发的JWT字符串" }`

### 安全
- 私钥通过环境变量注入，不写在代码里
- 身份核验现在是模拟的，生产环境需要对接真实机构

## 3. 声明式 API 模块

### 做什么
用 K8s 风格管理凭证——创建、查看、吊销。

### 接口
| 方法 | 路径 | 作用 |
|------|------|------|
| GET | `/v1/vc/credentials` | 列出所有凭证 |
| POST | `/v1/vc/credentials` | 创建新凭证 |
| GET | `/v1/vc/credentials/{id}` | 查看某一个凭证 |
| DELETE | `/v1/vc/credentials/{id}` | 吊销凭证 |

### 存储
- 用 SQLite 存数据（简单）
- 预留接口，以后可以换成 PostgreSQL

## 4. 合规审计模块

### 做什么
记录每一次操作，谁在什么时间做了什么，形成不可篡改的审计链条。满足《个人信息保护法》的要求。

### 怎么做
- 每次 API 调用都记录一条日志
- 每条日志包含：上一条日志的哈希 + 本次操作内容
- 任何人无法单独篡改某一条记录（因为会破坏哈希链条）

### 接口
- `GET /v1/audit/logs` — 查询审计日志
- `GET /v1/audit/crl` — 查询被吊销的凭证列表

## 开发顺序

1. 可信签发模块 — 补全"签发-持有-验证"全链路，答辩时可演示完整流程

3. 合规审计模块 — 加分项，审计日志有真实数据可记录
4. 云原生交付模块 — 最后统一打包，写 docker-compose
